# Product

## Register

product

## Users

Microbiology laboratory personnel use this calculator while planning and documenting prepared media batches. They need a task-focused interface that turns container counts and media selections into clear preparation quantities, then produces a printable batch sheet suitable for lab records.

## Product Purpose

The application calculates media preparation requirements for defined laboratory media types and generates an SOP-style batch preparation sheet on the same page. Success means the user can choose a media type, enter a container count, calculate raw and rounded volumes, see powder and additive quantities, and print or copy instructions without reloading the page.

## Brand Personality

Professional, precise, calm. The interface should feel like dependable laboratory tooling: clear enough for repeated daily use, modern enough to avoid paper-form friction, and restrained enough that calculations and batch records remain the focus.

## Anti-references

This should not look like a generic SaaS landing page, a playful classroom exercise, a dense spreadsheet, or a decorative medical-device mockup. Avoid ornamental visuals, marketing copy, exaggerated gradients, and UI choices that make printable records harder to read.

## Design Principles

- Put calculation confidence first: inputs, formulas, warnings, and outputs must be easy to verify at a glance.
- Treat print as a primary workflow: the batch preparation sheet must read cleanly on paper and preserve record fields.
- Keep the app single-page and direct: no page reloads, no hidden wizard steps, no unnecessary navigation.
- Use laboratory restraint: quiet hierarchy, consistent controls, and clear units beat visual novelty.
- Surface exceptions clearly: zero or unusually low calculated values should be visible before a user prints a batch sheet.

## Accessibility & Inclusion

Target WCAG AA contrast for text and controls. The interface must be mobile-friendly, keyboard-operable, readable in print, and understandable without relying on color alone. Motion should be minimal and respect reduced-motion preferences.
